Таблица прерываний

Системные вызовы - это обращение к средствам ОС из выполняемых процессов. Эти средства ОС часто бывают скрыты от пользователей. Для обеспечения переносимости программ, работа в среде языков программирования в некоторой степени инвариантна по отношению к текущей операционной системе. Однако реализация алгоритмического языка для определенной операционной системы использует системные вызовы. Системный вызов реализуется с помощью прерываний.
По аналогии с операционной системой MS DOS организуем все системные вызовы через 21h прерывание. В регистре AH - задается № функции, в регистре Al - № подфункции. Передача дополнительных аргументов в системные функции оговаривается отдельно (табл. 2.2).

Таблица 2.2 - Прерывания для системных вызавов

Номер функцииНазвание обслуживанияПодфункции
9hСервис клавиатуры (драйвер) Al = 00h - чтение символа без эха
Al = 01h - чтение строки без эха
Al = 02h - чтение символа с эхом
Al = 03h - чтение строки с эхом
Al = 0Fh - получение кода ошибки
10hСервис монитора(драйвер) Al = 00h - вывод символа
Al = 01h - установить позицию курсора
Al = 02h - считать позицию курсора
Al = 03h - прокрутить экран вверх/вниз
Al = 05h - вывести строку на экран
Al = 06 - очистить экран
13hРабота с файлами (драйвер) Al = 00h -открытие/ создание файла
Al = 01h - закрытие файла
Al = 03h - чтение из файла
Al = 04h - запись в файл
Al = 05h - смена атрибутов файла
Al = 06h - синхронизация
Al = 07h - получения информации о файле
Al = 08h - позиционирование в файле
Al = 09h - проверка на конец файла
Al = FEh - удаление файла
12hРабота с дисками Al = 00h - позиционированиеголовок
Al = 01h - чтение кластера
Al = 02h - запись кластера
Al = 03h - проверка кластера
Al = 04h - чтение информации о диске
Al = 05h - определение свободного мета на диске
Al = 0Еh - форматирование дорожки
Al = 0Fh - форматирование сектора
5hРабота с памятью Al = 00h - выделить память
Al = 01h - освободить память
Al = 02h - общее количество памяти
Al = 03h - наибольший свободный блок памяти
Al = 04h - оптимизация памяти
1hУправление процессами Al = 00h- запуск процесса
Al = 01h - приостановить процесс
Al = 02h - продолжение выполнения процесса
Al = 03h - получение информации о процессе с заданным PID
Al = 04h - получение PID родительского процесса
Al = 05h - возврат управления ядру
Al = 09h - уничтожение процесса
11hРабота с таймером Al = 00h - установка задержки для процесса
Al = 01h - сбросить задержку для процесса
Al = 02h - текущее значение таймера
15hРабота с датой Al = 00h - чтение даты
Al = 01h - установка даты
14hРабота с внутренними переменными Ah = 00h - создать переменную
Ah = 01h - удалить переменную
Ah = 02h - задать значение переменной
Ah = 03h - получить значение переменной
17hРабота с флагами событий AH = 00h - установка флага ожидания события от внешнего устройства
AH = 01h - его удаление
