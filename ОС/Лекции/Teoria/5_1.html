Интерактивные команды - это команды, которые вводятся пользователем с терминала. Есть набор основных команд, таких как: начало/завершение сеанса работы, создание/удаление элементов файловой структуры, порождение/уничтожение процессов, копирование и перенос файлов, перенос системы, организация защиты. Так же имеется набор дополнительных команд: установка и модификация параметров процессов/ресурсов, информационные команды. Основные интерактивные команды приведены в таблице 3.1

Квадратные скобочки обозначают необязательное наличие параметров.

Рассмотрим типы параметров команд:

  • - A - атрибут это строка, содержащая символы "x", "s", "h", "r", "w", "-". Для того чтобы установить атрибут необходимо в соответствующей позиции поставить соответствующую букву, если атрибут нужно сбросить, то ставится "-". Например, строка "-rw-r--r-" указывает на то, что файл могут читать все, а писать в него только владелец файла;
    s --- файл является системным
    h --- файл является скрытым
    x --- выполнение владельцу
    r --- запись владельцу
    w --- чтение владельцу
    x --- выполнение для группы
    r --- чтение для группы
    w --- запись для группы
    x --- выполнение для остальных пользователей
    r --- чтение для остальных пользователей
    w --- запись для остальных пользователей
  • D - дата, имеет формат dd.mm.[yy]yy, где dd-день, mm-месяц, [yy]yy - год. Например, 22.05.1997;
  • F - обозначает имя файла, может содержать полный путь к файлу, а так же в имени файла может содержать символы * и ?. ?-один любой символ, *-любая последовательность символов. Например, edit.txt или с:\ss\jul???.$$$ ;
  • N - число, содержит цифры 0-9;
  • P - обозначает имя директории может включать полный путь к директории. Например, c:\gifformat\perm или juk.asm;
  • S - строка, содержащая любые символы и начинающаяся с буквы;
  • SW - ключи команды, состоят из одного символа, который расположен после символа "-". Например, -n или -r;
  • T - время, имеет формат hh:mm.ss.[cc], где hh-часы, mm-минуты, ss-секунды, cc-сотые доли секунды. Например, 18:34.03;
  • X - обозначает имя НМД. Имя состоит из одной буквы после которой следует двоеточие. Например, a: или c:.

Опишем принцип работы интерактивных команд:

  • LOGIN. Первым параметром задается входное имя пользователя. Если первый параметр отсутствует, то команда запрашивает имя пользователя. Затем проверяется есть ли такой пользователь в системе. Если нет, то команда просит повторить ввод имени. Если такой пользователь существует в системе, то запрашивается пароль пользователя. При вводе пароля он не отображается на экране. Если введенный пароль не соответствует паролю пользователя, который хранится в системе, то повторяется ввод имени пользователя. Если пароли совпадают, то происходит вход в систему, и пользователь может вводить другие команды. В ОС существует спец Root ;
  • LOGOUT. Параметров не имеет. Происходит выход из системы пользователя, который запустил команду. Все процессы пользователя уничтожаются. После этой команды пользователь может выключить компьютер или выполнить команду login;
  • CRDIR. Если команда имеет два параметра, то директория указанная в первом параметре переименовывается, и получает имя указанное во втором параметре. Пользователь может переименовать директорию если является ее владельцем (хозяином). Если команда запускается с одним параметром, то создается директория, если она еще не существует, с именем указанным в первом параметре. Пользователь, создавший директорию, становится ее владельцем;
  • DELDIR. Если пользователь является владельцем директории, то команда удаляет директорию, заданную первым параметром, если в этой директории нет файлов или поддиректорий, иначе выводится сообщение. Root может удалить любую директорию;
  • DEL. Если пользователь является хозяином файла, то с помощью команды del он может его удалить. Имя удаляемого файла указывается в первом параметре. Root может удалить любой файл;
  • COPY. С помощью этой команды пользователь может скопировать файл, указанный в первом параметре команды, в файл,указанный во втором параметре. Пользователь может копировать файл, если является его владельцем, или если относится к группе владельца, и файл имеет атрибут "чтение для группы";
  • RENAME. Переименовывает файл указанный в первом параметре в файл указанный во втором параметре. Пользователь может переименовывать файлы если является их владельцем;
  • CHMOD. Команда позволяет менять атрибуты файла. Root может изменять все атрибуты, остальные пользователи могут менять только атрибуты чтения, записи и выполнения;
  • USER. С помощью этой команды Root может изменять количество пользователей, имеющих доступ к системе. Остальным пользователям команда недоступна. При удалении пользователя из системы все его каталоги и файлы удаляються;
  • KILL. С помощью команды kill процессу (процессам), указанному во втором параметре, подается сигнал завершения, указанный в первом параметре. Пользователь может удалять только свои процессы или процессы своей группы. Root может удалять процессы всех пользователей.
  • DIR. Выводит содержимое директории указанной в параметре, если параметр отсутствует то выводится содержимое текущей директории. Ключи N, E, D, S позволяют выводить файлы директории в отсортированном виде (S-по размеру, D-по дате, N-по имени, E-по расширению). Если указан ключ P, то при заполнении экрана, вывод приостанавливается, до нажатия пользователем любой клавиши;
  • PS. Выводит список процессов пользователя. Если указан ключ G или A, то выводится список процессов группы или всех пользователей соответственно;
  • CH. С помощью этой команды можно осуществить переход в директорию заданную параметром;
  • PRIO. Позволяет изменять приоритет процессов;
  • CLS. Очищает экран;
  • DATE и TIME. С помощью этих команд можно узнать системную дату и время. Если эти команды запустит Root они выдадут приглашение на ввод новой системной даты или времени;
  • DF. Показывает количество свободного места на диске, указанном в параметре. Если команда запускается без параметров, то подсчитывается количество свободного места на текущем диске. По умолчанию количество свободного места выводится в байтах, а если указан ключ K, то в килобайтах;
  • FORMAT. Форматирует НМД указанный в параметре. Команда запущенная с ключем S устанавливает на форматируемом диске ОС;
  • SYS. Позволяет перенести ОС с одного НМД на другой. Первый параметр указывает диск на который нужно перенести ОС. Команду может выполнить Root и его группа;
  • EDIT. Мощный редактор файлов.

Если перед интерактивной командой поставить символ "&", то команда запустить в фоновом режиме, т.е. она начнет исполняться, а пользователь в это время может вводить другие команды.

Кроме команд ОС, всякая пользовательская программа, которая содержится в файле и может быть выполнена, тоже является командой.
